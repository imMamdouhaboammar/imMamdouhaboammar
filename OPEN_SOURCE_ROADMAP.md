# Open Source Contribution Roadmap

This is my public contribution map for GitHub.

The focus is not to contribute everywhere. The focus is to contribute where my background creates a clear edge:

```txt
AI agents
  + agent skills
  + marketing workflows
  + public documentation
  + practical tooling
  = useful open source contributions
```

## Positioning

I sit between marketing strategy and AI systems.

That means I can contribute in areas where many technical projects need clearer workflows, better user-facing documentation, practical marketing use cases, and agent skills that solve real work.

## Public portfolio strategy

### 1. Build the profile as a living portfolio

The GitHub profile should make it clear within a few seconds:

- Who I am
- What I build
- Why my mix of marketing and AI systems matters
- What public projects are worth opening first
- Which communities I am trying to contribute to

### 2. Pin projects that explain the direction

The pinned projects should represent the system, not just random repositories.

Current focus:

- Riftbook: public field guide
- unslop-preflight: frontend quality gate
- agent-kernel: local memory and safety layer
- delegate-team: task routing and agent workflow layer

### 3. Make contributions visible and useful

Good contribution types:

- Small pull requests with clear scope
- README improvements that make a project easier to understand
- Issue triage with practical reproduction steps
- Agent skills with concrete examples
- Real workflow templates based on agency work
- Case studies that show how a tool is used in practice

### 4. Build around Agent Skills

Agent Skills are a strong fit because they match my background.

A good skill should include:

- Clear task boundary
- Required inputs
- Output format
- Review checklist
- Failure cases
- Example prompt
- Example result

Potential skill themes:

| Skill | Use case |
|---|---|
| Paid Media Plan Skill | Build campaign architecture, budget split, KPIs, creative testing, and tracking plan |
| Landing Page CRO Audit Skill | Review pages through offer clarity, friction, trust, CTA, and measurement gaps |
| Competitor Research Skill | Turn competitor signals into practical positioning and content angles |
| Content Repurposing Skill | Turn long-form material into posts, hooks, scripts, and content calendars |
| Agency Brief Cleaner Skill | Convert messy client notes into a clear brief, risks, questions, and next steps |

### 5. Use documentation as distribution

Every public contribution should be easy to understand from the README.

Useful README patterns:

- Explain the problem in plain language
- Show the first useful command or workflow quickly
- Add examples, not just features
- Mention who the project is for
- Make limitations visible
- Add contribution ideas for new contributors

## 30-day contribution plan

| Week | Focus | Output |
|---|---|---|
| Week 1 | Study target repos and open issues | Pick 2 repos, read contribution docs, identify 3 small fixes |
| Week 2 | Documentation and examples | Submit 1 or 2 README/example PRs |
| Week 3 | Agent skills | Create or contribute 1 marketing-agent skill with examples and review checklist |
| Week 4 | Case study and public post | Publish a short case study about the contribution and link it from the profile |

## Contribution rule

Do not chase noise.

Contribute where the work connects to the long-term direction:

```txt
AI agents that understand real marketing work
AI workflows that are clear enough to repeat
Agent skills that turn expertise into reusable systems
Public projects that teach while they build
```
